- `/structured/*.json` - JSON-LD endpoints for SEO

**Note:** `public/ai.txt` exists but is unused (dynamic route takes precedence).

## üì¶ Content Collections

### Schema Location
`src/content/config.ts` - Zod schemas for all collections

### Collections
1. **Projects** (`src/content/projects/`)
   - Schema: title, description, techStack, tags, featured, etc.
   - Format: Markdown with frontmatter

2. **Blog** (`src/content/blog/`)
   - Schema: title, description, category, tags, draft, etc.
   - Format: MDX with frontmatter
   - Categories: technology, ai, development, automation, creative, career, tutorial, finance

3. **Creative** (`src/content/creative/`)
   - Schema: title, description, mediaType, tools, category, etc.
   - Format: Markdown with frontmatter
   - Categories: ai-art, digital-art, experiment, commission

### Usage Pattern
```typescript
import { getCollection } from 'astro:content';
const projects = await getCollection('projects');
const featured = projects.filter(p => p.data.featured);
```

## üñºÔ∏è Image Handling

### Source of Truth
- **Imported images:** `src/assets/images/` (use `astro:assets` Image component)
- **Static images:** `public/images/` (use static URLs with explicit width/height)

### Best Practices
- Use `AImage` component for Astro pages
- Use static URLs with `width`, `height`, `loading`, `decoding` for React components
- Responsive presets: `src/lib/image-presets.ts` (hero, card, thumb)
- Formats: AVIF (primary), WebP (fallback)

## üîë Important Patterns

### Astro Islands
- Use `client:load` for above-the-fold interactive components
- Use `client:visible` for lazy-loaded components
- Minimize React usage (prefer Astro when possible)

### Component Organization
- Astro components: `.astro` files
- React components: `.tsx` files (in `components/`)
- UI primitives: `components/ui/` (ShadCN)

### Styling Approach
- Tailwind utility classes (preferred)
- Custom CSS in `src/styles/global.css` when needed
- Component-scoped styles in `<style>` blocks

### SEO Implementation
- Structured data: JSON-LD in `MainLayout` and `/structured/*.json`
- Meta tags: Dynamic via `MainLayout` props
- Sitemap: Auto-generated by `@astrojs/sitemap`

---

## üö® Common Gotchas

1. **Image imports:** Use `astro:assets` for imported images, static URLs for public images
2. **React hydration:** Only use React islands where interactivity is needed
3. **Content collections:** Always validate with Zod schemas in `config.ts`
4. **Build output:** Static files go to `dist/`, public files copied as-is
5. **Dynamic routes:** Use `getStaticPaths()` for collection-based routes
